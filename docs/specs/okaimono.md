# 仕様書 — おかいもの ゲーム (`okaimono.html`)

作成日: 2026-03-14

---

## 概要

コインを選んで商品の値段と「ぴったり」の金額を支払うゲーム。
お金の概念（硬貨の種類・足し算）を遊びながら体験する。

## 対象ユーザー

5歳前後の子ども。親がスマホを渡して一人で遊ぶ想定。

---

## 画面構成

```
┌──────────────────────────┐
│ [← もどる] 🪙おかいもの [Lv1⭐] │  ← ヘッダー
├──────────────────────────┤
│  🐻       🍬 あめ        │
│           ねだん: 2えん  │  ← ショップカウンター（木目調）
├──────────────────────────┤
│        ● ● ●            │  ← 進捗ドット（現在の問題番号）
├──────────────────────────┤
│  💰 はらうおかね         │
│  [1][1]                  │  ← 支払いトレイ（選んだコイン）
│  2えん        [← もどす] │
├──────────────────────────┤
│  👛 おさいふ             │
│  [1えん ×6]              │  ← 財布（タップでトレイへ）
└──────────────────────────┘
```

---

## ゲームフロー

```
startLevel()
  ↓
loadItem()  ← 商品表示・財布リセット
  ↓
コインをタップ → トレイに追加
  ↓
合計 == 値段？ → 成功モーダル → advance()
  ↓ (全商品クリア)
次のレベルへ or 全クリア
```

## 機能要件

### コイン操作
- 財布のコインをタップ → 支払いトレイに追加（財布の枚数 -1）
- トレイのコインをタップ → 財布に戻す（財布の枚数 +1）
- 「← もどす」ボタン → トレイの最後のコイン1枚を財布に戻す
- 財布の残枚数が 0 のコインは非活性（タップ不可・薄表示）

### 判定
- 合計 == 値段 → 自動で成功判定（0.35秒後にモーダル表示）
- 合計 > 値段 → 合計表示を赤にしてシェイク、フィードバック文言表示
- 各商品ごとに財布をリセット（前の問題のコイン選択は引き継がない）

### 進捗
- 進捗ドットでレベル内の何問目かを表示（完了：黄・現在：オレンジ・未来：グレー）

### 成功モーダル
| 状況 | 絵文字 | メッセージ | ボタン |
|---|---|---|---|
| 途中の問題 | 🎉 | ぴったり！すごい！ | つぎへ ▶ |
| レベルラスト問題 | 🎊 | レベル クリア！⭐ | つぎのレベルへ ▶ |
| 全クリア | 🏆 | ぜんぶ クリア！すごい！！ | はじめから ▶ |

### 演出
- 成功時に紙吹雪（🎉⭐🌟✨🎊💛💚💙🪙）を14個散らす
- ショップキャラクターがレベルごとに変わる（🐻🐰🐱🦊）
- 商品とショップキャラクターが上下に浮遊するアニメーション

## コイン デザイン

| 種類 | CSSクラス | 色 |
|---|---|---|
| 1円 | `.c1` | シルバーグラデーション |
| 5円 | `.c5` | ブロンズ/ゴールドグラデーション |
| 10円 | `.c10` | 銅色グラデーション |
| 50円 | `.c50` | シルバーグラデーション |
| 100円 | `.c100` | シルバーグラデーション |

財布のコイン：58px 丸型ボタン（ラベル＋残枚数バッジ付き）
トレイのコイン：42px 丸型チップ（タップで返却）

---

## デザイン要件

- 背景：温かみのある黄色オレンジグラデーション
- ショップカウンター：木目調ダークブラウン（`border-radius: 22px`）
- タップターゲット：最低 44px
- フォント：`Hiragino Maru Gothic Pro` 系
- `100dvh` + `env(safe-area-inset-bottom)` でスマホ対応
- `touch-action: manipulation` でダブルタップズーム無効
